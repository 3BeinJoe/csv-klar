- [x] Repository pushen und GitHub Pages aktivieren
- [x] veröffentlichte URL in Posting-Paket und CTAs eintragen
- [ ] Feedbackformular mit einem echten Test-Issue prüfen
- [ ] falls in Deutschland geschäftsmäßig betrieben: Impressumspflicht prüfen und erforderliche Anbieterangaben ergänzen
- [ ] Datenschutzhinweise für Hosting, Plattform-Links und späteres Analytics prüfen

## Alpha-Ziel

- fünf beobachtete Nutzertests
- mindestens drei echte Dateien ohne sensible Inhalte
- wiederkehrende Probleme dokumentieren
- keine Pro-Funktion entwickeln, bevor mindestens zwei Personen dasselbe Problem nennen

## Social-Test

- zehn Beiträge veröffentlichen
- Link-Klicks und tatsächliche Werkzeugnutzung wichtiger gewichten als Views
- keine bezahlte Werbung in der Alpha-Phase
- nach Beitrag zehn über Fortsetzung, Positionierungswechsel oder Stopp entscheiden
